# Security Policy

## 지원 버전

현재 보안 업데이트가 지원되는 버전:

| 버전 | 지원 상태 |
|------|-----------|
| 1.x  | ✅ 지원 중 |

## 취약점 보고

보안 취약점을 발견하셨다면, **공개 이슈로 보고하지 마세요.**

### 보고 방법

1. 이메일로 보고해 주세요
2. 제목에 `[SECURITY]`를 포함해 주세요
3. 다음 정보를 포함해 주세요:
   - 취약점 설명
   - 재현 방법
   - 영향 범위
   - 가능하다면 수정 제안

### 응답 시간

- 최초 응답: 48시간 이내
- 상태 업데이트: 7일 이내
- 수정 배포: 심각도에 따라 결정

### 보안 업데이트

보안 수정사항은 다음과 같이 배포됩니다:

1. 패치 버전으로 즉시 배포
2. CHANGELOG에 보안 수정 명시
3. 필요시 보안 권고(Security Advisory) 발행

## 보안 모범 사례

### 사용자 입력 처리

```tsx
// ❌ 위험: dangerouslySetInnerHTML 사용 자제
<div dangerouslySetInnerHTML={{ __html: userInput }} />

// ✅ 안전: 텍스트로 렌더링
<div>{userInput}</div>
```

### 의존성 관리

```bash
# 정기적으로 보안 취약점 검사
npm audit

# 취약점 자동 수정
npm audit fix
```

## 감사의 말

보안 취약점을 책임감 있게 보고해 주신 분들께 감사드립니다.
